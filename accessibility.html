Accessibility

We strive to meet WCAG level A and AA standards as laid out by W3C. Depending on project requirements, this is not always possible but we should ensure the client is aware of the failures to meet this standard and the impact this has to users.

Code validity

  • To allow assistive technologies to parse your code, it should always be tested against the W3C, to ensure the markup is well formed
  • Code should not error

Context

  • Provide context by using good semantic markup
  • Provide text alternatives for non-text content e.g videos and images
  • Meaningful content should be not be added to the dom via JavaScript
  • Even If missing from designs, semantic headings should be included where necessary and positioned off screen (not using display:none)

Logical flow

  • Present content in a meaningful order (Turn off stylesheets to check this)
  • Allow users to navigate using the keyboard through the site in a meaningful tab order

Images

  • Only populate alt tags where an image conveys essential information
  • Blank alt tags should be used for superfluous content to allow screen readers to skip past them
  • Don’t use images of text

Link

  • <a> tags should not be used for triggering JavaScript events. All links should have a destination
  • All links should have text content (alt tags on linked images counts as link text)
  • Links to different destinations should have unique text. ‘Read more’ is not sufficient
  • Links should have clear focus states
